P0 音效 - 实现方案选择

音效是"手感"的核心。三种可行方案：

A

Web Audio 合成增强

基于现有 AudioFeedback.ts 框架，用高质量合成算法替代简单蜂鸣：多层振荡器 + 噪声发生器 + 包络控制，模拟真实枪声的"砰"感。无需额外文件，体积为零，但上限受限于合成算法复杂度。

Pros

  • 零依赖，零文件体积
  • 可动态调参（距离衰减 / 室内混响）
  • 代码即音效，无版权问题

Cons

  • 永远达不到录音级别
  • 开发调参工作量大
  • 缺少枪声的金属撞击感
B

免版税音效库

从 Freesound.org / Mixkit / OpenGameArt 等开源音效库精选高品质枪声、脚步、弹壳落地声。约 30-50 个文件，总计 3-8 MB。脚步声可按材质分类（混凝土/沙地/金属/木头）。

Pros

  • 接近真实录音质量
  • 材质脚步声可实现
  • 文件小，加载快

Cons

  • 需要筛选匹配素材
  • 不同来源音质不一
  • 可能找不到完全匹配的枪型
C

混合方案（推荐）

核心高频音效（枪声/爆头/换弹）用免版税素材确保品质。低频氛围音（脚步/环境/弹壳）用 Web Audio 合成减少体积。约 15-20 个音频文件，总计 2-4 MB，按武器类型共享基础音色+参数变调区分每把枪。

Pros

  • 关键音效品质有保障
  • 文件少，管理简单
  • Web Audio 合成提供动态性

Cons

  • 两种系统并存，实现稍复杂
  • 仍需筛选部分素材
